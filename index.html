📖 项目概览

SharpConfig 是一个面向 .NET 平台的 CFG/INI 配置文件操作类库，由 Cem Dervis 创建并维护。 你可以使用它以文本或二进制格式读取、修改和保存配置文件及流。该库完全兼容 .NET、.NET Core 以及 Mono Framework，并且基于 .NET Standard 2.0 构建，具备出色的跨平台能力，可在 Windows、Linux、macOS 以及 Docker 环境中无缝运行[reference:0][reference:1]。

在日常 .NET 项目开发中，操作 INI/CFG 配置文件往往需要调用 WinAPI 或自行编写繁琐的解析逻辑，数据类型转换也是一大痛点。 SharpConfig 的核心目标就是解决这些烦恼：它提供了一套简洁直观的 API，让你能够以面向对象的方式操作配置文件，并自动处理类型转换[reference:2]。

🚀 快速入门

假设我们有如下配置文件 sample.cfg：

ini # 这是一个注释 [General] SomeString = Hello World! SomeInteger = 10 SomeFloat = 20.05 SomeBoolean = true SomeArray = { 1, 2, 3 } Day = Monday [Person] Name = Peter Age = 50

读取配置

csharp var config = Configuration.LoadFromFile("sample.cfg"); var section = config["General"]; string someString = section["SomeString"].StringValue; int someInteger = section["SomeInteger"].IntValue; float someFloat = section["SomeFloat"].FloatValue; bool someBool = section["SomeBoolean"].BoolValue; int[] someIntArray = section["SomeArray"].IntValueArray; DayOfWeek day = section["Day"].GetValue<DayOfWeek>();

内存中创建配置

csharp var myConfig = new Configuration(); // 自动创建 Section 和 Setting myConfig["Video"]["Width"].IntValue = 1920; myConfig["Video"]["Height"].IntValue = 1080; myConfig["Video"]["Formats"].StringValueArray = new[] { "RGB32", "RGBA32" }; myConfig["Video"]["Fullscreen"].BoolValue = true;

保存配置

csharp myConfig.SaveToFile("myConfig.cfg"); // 保存为文本文件 myConfig.SaveToBinaryFile("myConfig.cfg"); // 保存为二进制文件 myConfig.SaveToStream(myStream); // 保存到文本流 myConfig.SaveToBinaryStream(myStream); // 保存到二进制流
📌 重要行为说明：config["SectionName"] 和 section["SettingName"] 是"获取或创建"（create-or-get）API。 如果请求的项不存在，SharpConfig 会自动创建并返回，不会抛出异常。这一特性使得动态构建配置变得极为简便[reference:18]。

SharpConfig 提供了丰富的强类型属性，方便直接获取各种数据类型的配置值，无需手动转换。

5.2 数组类型

数组在配置文件中使用花括号 { } 语法定义，元素之间用逗号分隔。SharpConfig 会自动解析并转换为对应类型的数组[reference:19]。

csharp // 设置数组值 myConfig["Section"]["IntArray"].IntValueArray = new[] { 1, 2, 3 }; myConfig["Section"]["StrArray"].StringValueArray = new[] { "A", "B", "C" }; // 使用泛型方法获取任意类型数组 int[] intValues = myConfig["Section"]["IntArray"].GetValueArray<int>(); float[] floatValues = myConfig["Section"]["IntArray"].GetValueArray<float>();

5.3 枚举类型

SharpConfig 可直接将配置值解析为枚举类型，只需确保配置文本与枚举名称匹配即可[reference:20]。

csharp // 配置文件中: Day = Monday DayOfWeek day = config["General"]["Day"].GetValue<DayOfWeek>(); // day == DayOfWeek.Monday

🔄 对象映射

SharpConfig 支持将 Section 自动映射为 C# 对象，也支持将 C# 对象序列化为 Section。这是其最强大的特性之一，极大简化了配置与业务模型之间的转换[reference:21][reference:22]。

6.1 Section → 对象

假设我们有如下 Person 类和对应的 Section：

csharp public class Person { public string Name { get; set; } public int Age { get; set; } } // 从 Section 创建对象 Person person = config["Person"].ToObject<Person>(); Console.WriteLine($"{person.Name} is {person.Age} years old.");

6.2 对象 → Section

csharp var person = new Person { Name = "Alice", Age = 30 }; // 从对象创建 Section 并添加到配置中 var section = Section.FromObject("Person", person); config.Add(section); // 保存到文件 config.SaveToFile("person.cfg");

⚙️ 全局选项配置

某些项目有特殊的配置文件格式需求。SharpConfig 通过 Configuration 类的静态属性提供了灵活的行为定制[reference:53]。

选项 类型 说明
CultureInfo CultureInfo 值转换时使用的区域文化信息，默认为 InvariantCulture[reference:54]
ValidCommentChars char[] 有效的注释分隔字符数组，默认为 { '#', ';' }[reference:55]
PreferredCommentChar char 保存配置时使用的首选注释字符，默认为 '#'[reference:56]
ArrayElementSeparator char 数组元素分隔符，默认为 ','[reference:57]
IgnoreInlineComments bool 是否忽略行内注释（同一行中配置值后面的注释），默认为 false[reference:58]
IgnorePreComments bool 是否忽略前置注释（Section 或 Setting 上方的注释块），默认为 false[reference:59]

自定义注释与分隔符示例

csharp // 修改注释字符为 ';' Configuration.PreferredCommentChar = ';'; Configuration.ValidCommentChars = new[] { ';', '#' }; // 修改数组分隔符为 '|' Configuration.ArrayElementSeparator = '|'; // 忽略所有行内注释 Configuration.IgnoreInlineComments = true;
⚠️ 编码提示：如果配置文件包含中文或非 ASCII 字符，建议在加载时显式指定编码，否则可能出现乱码。 例如：Configuration.LoadFromFile("example.ini", Encoding.GetEncoding("gb2312"))。 同样，保存时也务必使用相同的编码[reference:60]。

🏁 总结

SharpConfig 是一个设计精巧、使用简便的 .NET 配置管理库。它用极小的体积和零依赖的代价，提供了完整的 CFG/INI 文件读写能力， 并在此基础上扩展了强类型访问、数组支持、对象映射等实用功能。无论是小型工具还是大型企业应用，SharpConfig 都能胜任配置管理的需求。

其核心优势可以概括为：

  • 极简 API：索引器 + 强类型属性，一目了然
  • 零依赖：不引入任何第三方包，减少项目复杂度
  • 全平台兼容：基于 .NET Standard 2.0，跨所有 .NET 运行时
  • 灵活可定制：注释字符、分隔符、文化信息均可配置
  • 开源免费：MIT 许可证，可自由使用和修改
